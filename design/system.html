Liz Performance App

Design System

A minimal, opinionated design system for a performance tracking PWA. Five-AM-proof — every action takes one thumb, every screen loads instantly, every input is reachable without scrolling. This is a living document, updated as each tab is designed and built.

Overview

Principles

Every design decision runs through these four lenses. If a UI element fails one, revisit it.

bolt
Five-AM-Proof
Works at 5 AM with one hand. No friction. Tap in, log it, go lift.
fitness_center
Performance Aesthetic
Bold type, high contrast, functional color. Strong, not decorative. Data reads first.
grid_view
System Thinking
Consistent patterns, reusable components. Nothing one-off. Everything tokens.
favorite
Athlete Context
Postpartum athlete, breastfeeding. Every decision respects the full picture.

Patterns

Session flow

Six screen states in a complete workout session. Each state has one primary action and no decision fatigue.

State 01
Pre-session
Exercise list preview. Single CTA: Start Session.
State 02
Active
Expanded exercise. Set rows. Live timer. Progress bar.
State 03
Paused
Dimmed content. Bottom sheet: Resume or End.
State 04
Rest timer
Dark overlay. Countdown sheet. Skip Rest.
State 05
Complete
Mint check. Stats. PR badge. Log This Session CTA.
State 06
Rest day
Empty state. Next session preview. View Program.
